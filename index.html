✦ The core finding — offline accuracy does not predict closed-loop success

Best-in-study offline prediction, zero task completion. OpenVLA achieves the lowest offline action-prediction error of any model we evaluated (1.0% normalized MAE, comparable to GR00T N1.6) — yet completes 0% of trials. It reaches and grasps reliably, then freezes.

Why it freezes: OpenVLA is stateless and single-step — it predicts one action from the current image only, with no proprioceptive feedback and no temporal action context. Once the brush is grasped, the scene becomes visually near-static (the brush is occluded in the gripper), so the policy sees almost no change and emits almost no motion. The extended post-grasp transport phase provides none of the visual dynamics this architecture depends on.

How we deployed OpenVLA in closed loop

OpenVLA is a single-system autoregressive policy: given one image and a language instruction, it directly predicts a discretized action token sequence — no diffusion head, no proprioceptive state, no action chunking.

Architecture & inference

  • Base: OpenVLA-7B (Prismatic VLM)
  • Vision: fused SigLIP + DINOv2
  • LLM: Llama-2 7B
  • Action: 256 discrete bins / dim, autoregressive
  • State input: none (image + language only)
  • Output: single 19-D action (T=1)

Fine-tuning

  • Method: LoRA (r=32, α=16) on the LLM
  • Vision encoders: frozen
  • Steps: 100,000 · batch 8
  • Hardware: NVIDIA DGX Spark
  • Data format: RLDS (converted from LeRobot v2.1)
  • Attention: eager (FlashAttention disabled)

Single-step, synchronous inference bridge

Unlike the chunked diffusion policies (GR00T, StarVLA), OpenVLA returns one action per query. The bridge queries synchronously and holds each action for 5 control steps at 10 Hz, so every action is conditioned on the current observation — never a stale one.

python
# OpenVLA: single-step, synchronous, no chunking, no EMA
action = openvla_client.get_action(
    image=cam_image_224,          # 224x224 RGB, no color conversion
    instruction=task_description,
    unnorm_key="ffw_sg2",   # per-dim unnormalization stats
)                                 # returns one 19-D action (T=1)
# held for 5 control steps at 10 Hz (~500 ms) before re-query

Gripper-snap at inference

Raw gripper output is soft; a decisive open/close snap is applied so the grasp actually closes on the handle. The 100% grasp rate depends on this post-processing.

python
# decisive gripper command from raw model output
rg = 1.1 if raw_gripper > 0.3 else 0.05

Why it reaches and grasps but cannot place

During reach, the scene changes continuously as the arm approaches — the single-step policy always has fresh visual signal to act on. After the grasp, the brush is held in the gripper and the scene stops changing; a stateless, single-step model sees a near-identical image at every step and therefore emits a near-identical (near-zero-motion) action. With no proprioceptive feedback and no temporal action context, there is nothing to carry the trajectory through the visually static transport phase.

Offline action prediction error across all ten training checkpoints (10k–100k) — evaluated on 10 dataset episodes × 30 frames (300 samples each) without simulation. Action range mean 0.8441 rad.

MAE · RMSE · Normalized MAE vs training steps (left); overall MAE per checkpoint (right)

Key finding — a clean, converged offline curve

Offline MAE decreases monotonically from 0.0234 rad (2.6% NormMAE) at 10k to 0.0101 rad (1.0% NormMAE) at 100k, with no collapse or instability — unlike StarVLA. The deployed 100k checkpoint is also the global offline minimum. That a smooth, well-converged, best-in-study offline curve still yields 0% closed-loop place success is the strongest evidence in our study that offline MAE does not predict closed-loop task completion.

Start the OpenVLA server & GUI

bash
# OpenVLA policy server (HTTP, port 8000)
python vla-scripts/deploy.py --openvla_path /path/to/spark-100k \
    --host 0.0.0.0 --port 8000

# control GUI — open http://localhost:8000
python scripts/server.py

The GUI writes instructions to /tmp/groot_task.txt, polled by the inference loop each step. OpenVLA runs single-step at 10 Hz. Special commands __RESET__ and __STOP__ control the environment without interrupting simulation.
